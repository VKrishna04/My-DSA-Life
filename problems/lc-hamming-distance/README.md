## Problem Statement

The Hamming distance between two integers is the number of positions at which the corresponding bits are different.

Given two integers `x` and `y`, return _the **Hamming distance** between them_.

 

**Example 1:**

**Input:** x = 1, y = 4
**Output:** 2
**Explanation:**
1   (0 0 0 1)
4   (0 1 0 0)
       &uarr;   &uarr;
The above arrows point to positions where the corresponding bits are different.

**Example 2:**

**Input:** x = 3, y = 1
**Output:** 1

 

**Constraints:**

	- `0 <= x, y <= 231 - 1`

 

**Note:** This question is the same as  2220: Minimum Bit Flips to Convert Number.

## Solutions

```Python3
class Solution:
    def hammingDistance(self, x: int, y: int) -> int:
        return bin(x ^ y)[2:].count("1")

```

## AI Review

### 1. Complexity
*   **Time Complexity:** $O(W)$, where $W$ is the number of bits (at most 31 for this problem). The `bin()` conversion and `.count()` both iterate over the bit-string.
*   **Space Complexity:** $O(W)$ to store the intermediate binary string generated by `bin()`.

### 2. Correctness
The solution is **correct**. It leverages the property that XOR (`^`) outputs `1` only where bits differ. Slicing `[2:]` is technically optional as `count("1")` ignores the `0b` prefix anyway. It handles the edge case $x = y$ (returning 0) and the maximum constraint $2^{31}-1$ perfectly.

### 3. Concrete Optimization
Use the built-in **`int.bit_count()`** method (available in Python 3.10+):
```python
return (x ^ y).bit_count()
```
**Why:** This eliminates the overhead of string allocation, slicing, and character comparison. It maps to highly optimized CPU instructions (like `POPCNT`), reducing space complexity to **$O(1)$**.

### 4. Key Algorithmic Pattern
**Bit Manipulation:** Specifically using **XOR** to identify differing bits followed by a **Population Count** (counting set bits).
