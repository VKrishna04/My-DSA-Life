class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        # max_length = 1
        # if s is "":
        #     return 0
        # for i in range(len(s)):
        #     st = ""

        #             st += ch
        #             max_length=max(max_length,len(st))
        #         else:
        #             break
        # print(max_length)
        # return max_length

        start = 0  
        max_length = 0  
        seen_chars = set()  

        for end in range(len(s)):
            while s[end] in seen_chars:
                seen_chars.remove(s[start])
                start += 1
            
            seen_chars.add(s[end])
            
            max_length = max(max_length, end - start + 1)

        return max_length

### Assistant

This solution uses the **Sliding Window** technique with **Two Pointers**. 

Think of it like a flexible "window" that moves across the string. The window expands to include new characters but shrinks whenever it encounters a duplicate.

### How it works step-by-step:

1.  **Initialization**:
    *   `start`: The left boundary of your window.
    *   `end`: The right boundary (controlled by the `for` loop).
    *   `seen_chars`: A **Set** that tracks which characters are currently inside your window. Sets are used because checking for existence (`in`) takes constant time ($O(1)$).

2.  **Expanding the Window (The `for` loop)**:
    *   The `end` pointer moves forward character by character. 

3.  **Handling Duplicates (The `while` loop)**:
    *   Before adding `s[end]` to the set, the code checks: *"Is this character already in our window?"*
    *   If **Yes**: The window is no longer "unique." To fix this, we shrink the window from the left by:
        1.  Removing `s[start]` from the set.
        2.  Moving the `start` pointer forward.
    *   We repeat this until the duplicate character is gone.

4.  **Updating the Result**:
    *   `seen_chars.add(s[end])`: Now that we've cleared duplicates, we safely add the new character.
    *   `max_length = max(max_length, end - start + 1)`: We calculate the current window size (`end - start + 1`) and update our record if it’s the longest we've seen so far.

### Why is this better than your commented-out code?

*   **Your old code ($O(n^2)$):** For every single character, you started a brand new search. If the string was 50,000 characters long, this would be too slow.
*   **This code ($O(n)$):** Each character is visited by the `end` pointer once and the `start` pointer once. Even though there is a `while` loop inside a `for` loop, the `start` pointer only ever moves forward, never backward. This makes it extremely efficient for large inputs.
